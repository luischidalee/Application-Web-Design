Markdown  
---------------
Los archivos en Markdown se guardan con la extensión .md y se pueden abrir en un editor de texto como TextEdit, Notepad, Sublime Text o Vim. Muchos sitios web o plataformas de publicación también ofrecen editores basados en la web y/o extensiones para introducir texto utilizando la sintaxis de Markdown.

Markdown  Opciones de etiquetado
---------------
1. Encabezados: Markdown dispone de cuatro niveles de encabezados definidos por el número de # antes del texto del encabezado. Pega los siguientes ejemplos en la caja de texto de la izquierda:
	* # Primer nivel de encabezado
	* ## Segundo nivel de encabezado
	* ### Tercer nivel de encabezado
	* #### Cuarto nivel de encabezado
2. Añadir énfasis: El texto se puede poner en cursivas encerrándolo entre los símbolos * o -. De la misma forma, el texto en negritas se escribe encerrando la palabra entre ** o __ .
	* ¡Estoy **muy** entusiasmado con los tutoriales de _The Programming Historian en español_!
3. Listados: Markdown soporta la creación de listas ordenadas y sin ordenar.
	* Lista de compras
		---------------
	* Frutas
 	 * Manzanas
 	 * Naranjas
 	 * Uvas
	* Lácteos
	  * Leche
	  * Queso

	* Las listas ordenadas se escriben numerando cada línea. Una vez más, el objetivo de Markdown es producir documentos que sean legibles como texto plano y que a la vez puedan traducirse a otros formatos.

Comandos de GIT
---------------
1. **git status, git tag, git blame** : Comprobar el estado de un repositorio local.
2. **git init, git clone, git config** : Agregue archivos individuales o globalmente.
3. **git add, git commit, git diff, git stash, .gitignore** : Agregar comentarios a la confirmación.
4. **git remote, git fetch, git push, git pull** : Cargue sus cambios en el repositorio remoto.
5. **git branch, git checkout, git merge** : Cree, explore y elimine branches.
6. **git checkout, git clean, git revert, git reset, git rm** : Revertir un repositorio a una confirmación específica.
